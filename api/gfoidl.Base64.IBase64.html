Decode(ReadOnlySpan<Byte>, Span<Byte>, out Int32, out Int32, Boolean)

Base64 decodes encoded.

Declaration
OperationStatus Decode(ReadOnlySpan<byte> encoded, Span<byte> data, out int consumed, out int written, bool isFinalBlock = true)
Parameters
Type Name Description
System.ReadOnlySpan<System.Byte> encoded

The base64 encoded data.
System.Span<System.Byte> data

The base64 encoded data to decode.
System.Int32 consumed

The number of input bytes consumed during the operation. This can be used to slice the input for subsequent calls, if necessary.
System.Int32 written

The number of bytes written into the output span. This can be used to slice the output for subsequent calls, if necessary.
System.Boolean isFinalBlock

true (default) when the input span contains the entire data to decode. Set to false only if it is known that the input span contains partial data with more data to follow.
Returns
Type Description
System.Buffers.OperationStatus

It returns the OperationStatus enum values:

  • Done - on successful processing of the entire input span
  • DestinationTooSmall - if there is not enough space in the output span to fit the decoded input
  • NeedMoreData - only if isFinalBlock is false and the input is not a multiple of 4, otherwise the partial input would be considered as InvalidData
  • InvalidData - if the input contains bytes outside of the expected base64 range, or if it contains invalid/more than two padding characters, or if the input is incomplete (i.e. not a multiple of 4) and isFinalBlock is true.
Exceptions
Type Condition
System.FormatException

Thrown for Url when the length is not conforming the base64Url standard. isFinalBlock set to false won't throw this exception.

Decode(ReadOnlySpan<Char>)

Base64 decodes encoded into a System.Byte array.

Declaration
byte[] Decode(ReadOnlySpan<char> encoded)
Parameters
Type Name Description
System.ReadOnlySpan<System.Char> encoded

The base64 encoded data in string-form.
Returns
Type Description
System.Byte[]

The base64 decoded data.
Remarks

For base64 decoding on .NET Full or .NET Standard perf-wise it may be better to use System.Convert.FromBase64String(System.String) or System.Convert.FromBase64CharArray(System.Char[],System.Int32,System.Int32) if possible.
Please benchmark yourself.

base64Url or .NET Core is not affected by this.

Exceptions
Type Condition
System.FormatException

The input is not a valid Base64 string as it contains a non-base 64 character, more than two padding characters, or an illegal character among the padding characters.
See Also
https://github.com/gfoidl/Base64/issues/137

GetMaxDecodedLength(Int32)

Gets the maximum length of the decoded data. The result may not be the exact length due to padding. Use GetDecodedLength(ReadOnlySpan<Byte>) or GetDecodedLength(ReadOnlySpan<Char>) for an accurate length.

Declaration
int GetMaxDecodedLength(int encodedLength)
Parameters
Type Name Description
System.Int32 encodedLength

The length of the encoded data.
Returns
Type Description
System.Int32

The maximum base64 decoded length of encodedLength.
Remarks

This method can be used for buffer-chains, to get the size which is at least required for decoding.

Exceptions
Type Condition
System.ArgumentOutOfRangeException

Thrown when encodedLength is negative.
